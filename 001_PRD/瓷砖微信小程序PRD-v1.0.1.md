## 1. 产品概述

### 1.1 产品定位

一款面向家装业主、装修工人/包工头、设计师/装修公司三端用户的品牌自营瓷砖微信小程序。以高质量线上展示解决"看不见实物不敢买"的核心痛点，打通从浏览到下单的完整购买链路。

### 1.2 目标用户

| 用户角色 | 特征 | 核心诉求 |
|----------|------|----------|
| 家装业主 | 装修自己房子的普通消费者 | 看清产品细节和铺贴效果，获取导购建议 |
| 装修工人/包工头 | 替业主采购瓷砖 | 快速查找产品，高效下单 |
| 设计师/装修公司 | 为客户选材 | 获取高清素材和产品规格信息 |

### 1.3 核心价值

**让用户在线上就能看清楚瓷砖，敢下单。**

- 高清产品图 + 纹理细节放大
- 多场景真实装修效果图
- 完整规格参数透明展示

## 3. 用户注册与登录（新增）

### 3.1 登录流程

微信小程序使用微信官方授权体系，无需用户设置密码。

**首次登录流程：**

```
用户打开小程序
    │
    ▼
检测登录状态
    │
    ├── 已登录 ──────────── 直接进入首页
    │
    └── 未登录
          │
          ▼
    ┌──────────────┐
    │  欢迎页/授权页  │
    │  一键微信登录   │
    └──────┬───────┘
           │ 用户点击"微信授权登录"
           ▼
    wx.login 获取临时 code
           │
           ▼
    后端用 code 换取 openid / session_key
           │
           ├── 老用户（openid 已存在）── 生成 token → 进入首页
           │
           └── 新用户（openid 不存在）
                 │
                 ▼
           ┌──────────────┐
           │   角色选择页   │
           │  选择你的身份   │
           │  · 家装业主    │
           │  · 装修工人    │
           │  · 设计师      │
           └──────┬───────┘
                  │
                  ▼
           ┌──────────────┐
           │  手机号绑定页  │
           │  获取微信手机号 │
           │  （可跳过）    │
           └──────┬───────┘
                  │
                  ▼
            创建用户 → 进入首页
```

**再次打开（已登录）：**

```
打开小程序 → 静默调用 wx.login → 后端校验 token → 直接进入首页
```

### 3.2 登录/授权页设计

**页面布局（从上到下）：**

| 区域 | 内容 | 说明 |
|------|------|------|
| 品牌区 | Logo + 品牌名 + Slogan | 居中展示，建立品牌认知 |
| 功能展示区 | 3 张轮播图 | 简要介绍小程序核心功能（看高清图/选瓷砖/下单支付） |
| 授权按钮区 | "微信授权登录"按钮 | 品牌主色大按钮，底部附隐私协议勾选 |
| 隐私协议 | 《用户协议》《隐私政策》链接 | 蓝色小字，首次需勾选同意 |

**隐私合规要点：**
- 用户未点击授权前，不调用任何隐私接口
- 隐私协议必须可点击查看完整内容
- 手机号授权单独弹窗，用户可拒绝（拒绝后仍可浏览商品，但下单时强制绑定）
- 用户可在个人中心随时修改角色、重新绑定手机号
- 用户有权在设置中注销账号（删除个人数据）

### 3.3 角色选择页设计

**页面布局（从上到下）：**

| 区域 | 内容 | 说明 |
|------|------|------|
| 标题 | "请选择你的身份" | 居中，配副标题"便于为你提供更好的服务" |
| 角色卡片 | 3 张大卡片纵向排列 | 每张卡片包含图标、角色名、一句话描述 |
| 确认按钮 | "确认" | 选中角色后高亮可点击 |

**角色卡片内容：**

| 角色 | 图标 | 描述 |
|------|------|------|
| 家装业主 | 🏠 | 装修自己的房子，需要看清产品细节和铺贴效果 |
| 装修工人/包工头 | 🔧 | 替业主采购瓷砖，需要快速找产品、高效下单 |
| 设计师/装修公司 | 🎨 | 为客户选材，需要高清素材和详细规格参数 |

### 3.4 手机号绑定页设计

**页面布局：**

| 区域 | 内容 | 说明 |
|------|------|------|
| 标题 | "绑定手机号" | 居中文案 |
| 说明 | "用于订单物流通知和售后联系" | 解释绑定目的，降低用户顾虑 |
| 授权按钮 | "微信手机号一键授权" | 调用微信 getPhoneNumber 组件 |
| 跳过 | "暂不绑定"链接 | 灰色小字，允许跳过 |

**业务规则：**
- 用户选择"暂不绑定"后正常进入首页
- 下单时若未绑定手机号，强制弹出手机号授权
- 手机号通过微信加密数据获取，后端使用 session_key 解密后存储
- 手机号存储需脱敏，前端仅显示 `138****6789` 格式

### 3.5 用户信息存储结构

| 字段 | 类型 | 说明 |
|------|------|------|
| 用户ID | string | 唯一标识，后端生成 UUID |
| openid | string | 微信 openid，每个小程序唯一 |
| unionid | string | 微信 unionid（如有），用于跨应用识别 |
| 昵称 | string | 微信昵称（用户授权后获取） |
| 头像 | string | 微信头像 URL（用户授权后获取） |
| 手机号 | string | 加密存储，前端脱敏展示 |
| 角色 | enum | 家装业主 / 装修工人 / 设计师，注册时选择，后续可修改 |
| 注册时间 | datetime | 首次登录时间 |
| 最近登录 | datetime | 最近一次登录时间 |
| 状态 | enum | 正常 / 已注销 |

## 4. 核心页面详述

### 4.1 产品详情页（重点）

这是整个小程序最关键的页面，直接决定用户是否下单。

**页面布局（从上到下）：**

| 区域 | 内容 | 说明 |
|------|------|------|
| 图片区 | 产品高清大图轮播 | 支持双指捏合放大，看清瓷砖纹理细节；3-5张产品图 |
| 基础信息 | 产品名称、型号、价格（元/片） | 价格醒目，支持显示"箱"为单位 |
| 规格参数 | 表格形式展示 | 尺寸、材质、工艺、表面处理、厚度、适用空间、风格、每箱片数 |
| 效果图 | 多场景装修实景图 | 客厅、厨房、卫生间等实际铺贴效果，辅助决策 |
| 底部操作栏 | 客服、购物车、立即购买 | 固定底部，始终可见 |

**交互细节：**
- 图片区双击放大到最大，再次双击恢复
- 效果图区域横向滑动浏览
- 点击"加入购物车"弹出规格确认弹窗（数量选择 + 确认按钮）
- 点击"立即购买"直接跳转下单页

### 4.2 首页

- 顶部搜索栏：支持产品名称、型号搜索
- Banner 区：2-3张轮播，展示新品/活动
- 分类入口：4个图标入口（按类型 / 按空间 / 按风格 / 按颜色）
- 推荐产品：双列瀑布流展示产品卡片（图 + 名称 + 价格）

### 4.3 购物车

- 按商家分组显示（当前为自营，无分组）
- 每项：勾选框 + 缩略图 + 名称 + 规格简述 + 单价 + 数量调节 + 小计
- 底部：全选 + 合计金额 + 去结算按钮
- 支持左滑删除商品

### 4.4 下单页

- 收货地址卡片（可展开选择/新增）
- 商品清单（不可编辑）
- 金额明细：商品总额、运费（显示"线下协商"）、合计
- 备注输入框
- 底部：合计金额 + 提交订单按钮
- 提交后唤起微信支付

### 4.5 订单管理（用户端）

| 状态 | 说明 | 用户可操作 |
|------|------|-----------|
| 待付款 | 已提交未支付 | 取消订单、去付款 |
| 待发货 | 已付款，商家处理中 | 联系客服 |
| 已完成 | 已发货/已送达 | 查看详情 |

### 4.6 个人中心（更新）

| 区域 | 内容 | 说明 |
|------|------|------|
| 头部 | 头像、昵称、角色标签 | 点击可修改角色 |
| 订单入口 | 待付款/待发货/已完成 | 显示各状态数量角标 |
| 功能列表 | 全部订单、收货地址、手机号 | 手机号显示脱敏，可点击修改 |
| 其他 | 联系客服、关于我们、注销账号 | 注销账号在底部，红色文字 |

### 4.7 视觉设计规范

**主色调：天蓝色系列**

| 色阶 | 色值 | 用途 |
|------|------|------|
| 品牌主色 | `#1890FF` | 按钮、导航栏、图标、选中态 |
| 浅蓝 | `#E6F7FF` | 背景色、卡片底色 |
| 悬浮蓝 | `#40A9FF` | 按钮悬浮、点击态 |
| 深蓝 | `#096DD9` | 重要文字、链接色 |
| 极浅蓝 | `#F0F5FF` | 页面全局背景 |

**辅助色**

| 色阶 | 色值 | 用途 |
|------|------|------|
| 价格红 | `#FF4D4F` | 价格文字、促销标签 |
| 成功绿 | `#52C41A` | 成功状态、已完成标签 |
| 警告橙 | `#FAAD14` | 待处理标签 |
| 文字黑 | `#262626` | 标题、正文 |
| 文字灰 | `#8C8C8C` | 辅助说明、次要信息 |
| 分割线 | `#F0F0F0` | 列表分割、卡片边框 |
| 白色 | `#FFFFFF` | 卡片背景、页面底色区域 |

**应用规则**

- 底部导航栏：选中态使用品牌主色 `#1890FF`，未选中灰色
- 主要操作按钮（立即购买、提交订单、微信授权登录）：品牌主色白字
- 次要操作按钮（加入购物车）：品牌主色描边，白色底
- 价格：使用价格红 `#FF4D4F`
- 商品卡片：白色背景，浅蓝边框，悬浮时加深阴影
- 订单状态标签：待付款-警告橙，待发货-品牌主色，已完成-成功绿

## 7. 非功能需求

### 7.1 性能

- 图片需压缩优化，首页首屏加载 < 2秒
- 产品图支持 CDN 加速
- 列表页分页加载，每页20条
- 登录鉴权耗时 < 500ms（静默登录）

### 7.2 兼容性

- 支持微信小程序基础库 2.10+
- 适配 iOS 和 Android 两端
- 支持 iPhone SE 至最新型号、主流 Android 机型

### 7.3 安全

- 用户手机号需授权获取，符合微信隐私规范
- 微信 openid 和 session_key 仅在服务端处理，不暴露给前端
- 手机号 AES 加密存储，前端仅展示脱敏格式
- 支付对接微信支付，不自行存储支付敏感信息
- 商家端需账号密码登录，支持权限控制
- 用户注销后个人数据按《个人信息保护法》要求删除

### 7.4 隐私合规

- 用户首次进入时展示隐私协议弹窗，需明确同意后才调用微信授权接口
- 手机号授权独立触发，不与登录授权捆绑
- 小程序设置页提供"隐私设置"入口，用户可查看已授权信息
- 提供账号注销功能（"我的"页面底部），注销后 30 天冷静期，到期后永久删除

### 9.2 微信登录接口说明

| 接口 | 说明 | 调用时机 |
|------|------|----------|
| `wx.login()` | 获取临时 code，换取 openid | 每次启动 |
| `wx.getUserProfile()` | 获取用户昵称、头像（需用户点击授权） | 登录页点击授权按钮 |
| `wx.getPhoneNumber()` | 获取加密手机号（需用户点击授权） | 注册流程/下单前 |
| `wx.checkSession()` | 检查 session_key 是否过期 | 启动时静默检查 |
